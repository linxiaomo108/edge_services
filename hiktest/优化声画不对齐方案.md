# 海康 NVR 视频分段下载音画不同步 —— 完整修复方案

> **环境说明**：服务端 Windows + Python，客户端网页浏览器（HTML5 video 标签）  
> **目标**：网页播放从 NVR 下载的多段合并视频，音画同步，画质清晰  
> **基于**：VSPlayer 安装包逆向、四个 SDK 头文件分析、ffmpeg.py 代码审查

> 基于对 VSPlayer 安装包、SystemTransform SDK 头文件、FileDemo.java 及 ffmpeg.py 的逆向与源码分析
>VSPlayer是海康的视频播放器，使用它去直接播放从NVR下载的视频，播放时声画是同步的，意味着这个播放器可以处理这类声画不同步的问题；
>ystemTransform是指咱们项目中使用的转封装库文件；
>ffmpeg.py是咱们项目中，视频下载后集中处理时的代码文件；

## 一、问题根因（已确认）

### 1.1 根本原因：转封装时缺少两个关键调用

你现在用的 `SystemTransform.dll`（转封装库）调用流程是：

```
SYSTRANS_Create → SYSTRANS_Start → 轮询 GetTransPercent → SYSTRANS_Stop → SYSTRANS_Release
```

**缺失的关键调用**：

| 缺失 API | 所在 DLL | 后果 |
|---|---|---|
| `FC_SetGlobalTime` | FormatConversion.dll（转码库） | 各分段 PTS 基准不统一，段间拼接时间轴断裂 |
| `FC_ReSetTimeStamp` | FormatConversion.dll（转码库） | 音频 PTS 独立漂移，与视频 PTS 越来越偏 |

> **重要澄清**：前面方案提到的 `SYSTRANS_ResetPackInfo`（`bTimestampSync=1`）经过对 `SystemTransform.dll` 实际导出函数的验证，**该函数不存在于你的 SystemTransform.dll 中**。正确的 API 在另一个库 `FormatConversion.dll`（转码库包）里。

### 1.2 为什么 VSPlayer 能直接播放，你的流程不能

VSPlayer 内部对每帧维护两套时间（从 PlayCtrl.dll 逆向出的日志格式）：

```
DecodeFrame Video || FrameType:%d FrameTime:%d AbsFrameTime:%d FrameNum:%d AbsFrameNum:%d
```

| 时间戳 | 来源 | VSPlayer 的用途 |
|---|---|---|
| `FrameTime` | MP4 容器层 PTS | 仅参考，可以是错的 |
| `AbsFrameTime` | HIK 私有帧头中的设备时钟 | **音视频同步的真正基准** |

VSPlayer 用 `AbsFrameTime` 对齐音视频，PTS 损坏与否无所谓。你的浏览器 `<video>` 标签只能读容器 PTS，所以必须在服务端把 PTS 修好再给浏览器播。

### 1.3 分段下载加剧问题的原因

```
NVR 录像（3 小时）
    ↓ 海康 SDK 按 1G 上限自动切分
    ↓ 得到 8～9 个分段，每段各自有一套 PTS（从 0 或任意值开始）
    ↓ 转封装时没有告知 SDK 该段的真实起始时刻
    ↓ 各段 PTS 基准不同，音频 PTS 独立漂移
    ↓ ffmpeg 合并时以容器 PTS 为准，段间出现时间轴跳变
    ↓ 浏览器播放：画面跳、音频提前/延迟、或完全不同步
```

### 第一步：用 FormatConversion.dll 替换转封装调用（根治方案）

这是最重要的一步，预计解决 80-90% 的不同步问题。

#### 3.1.1 为什么换 FormatConversion.dll

经过对实际 DLL 导出函数的验证，`FormatConversion.dll`（转码库 WIN64）明确包含：

```
FC_SetGlobalTime        ← 设置分段真实起始时刻
FC_ReSetTimeStamp       ← 强制音频时间戳对齐到视频
FC_RegisterDetailedCB  ← 逐帧回调，可拿到设备绝对时钟
```

而你现在用的 `SystemTransform.dll` 不含后两个。

#### 3.1.3 Python 封装代码（hik_convert.py）

```python
"""
hik_convert.py
使用 FormatConversion.dll 对 HIK 格式分段进行转封装。
关键：设置全局时间基准 + 强制音视频时间戳同步。
"""
import ctypes
import time
import logging
import os
import json
from datetime import datetime
from pathlib import Path

logger = logging.getLogger(__name__)


# ── 结构体定义（对应 FormatConversionDefine.h）─────────────────────────────

class FC_GLOBAL_TIME(ctypes.Structure):
    """设备绝对时钟（与 HIK NVR 的 struStartTime 格式一致）"""
    _fields_ = [
        ("sYear",       ctypes.c_uint16),
        ("sMonth",      ctypes.c_uint16),
        ("sDayOfWeek",  ctypes.c_uint16),   # 填 0 即可
        ("sDay",        ctypes.c_uint16),
        ("sHour",       ctypes.c_uint16),
        ("sMinute",     ctypes.c_uint16),
        ("sSecond",     ctypes.c_uint16),
        ("sMs",         ctypes.c_uint16),   # 毫秒
    ]


class FC_DETAILED_CB_INFO(ctypes.Structure):
    """逐帧回调信息（FC_RegisterDetailedCB 的回调参数）"""
    _fields_ = [
        ("nTrackIndex",           ctypes.c_uint32),
        ("nFrameTypeOrBPS",       ctypes.c_uint32),
        ("nWidthOrChannels",      ctypes.c_uint32),
        ("nHeightOrSampleRate",   ctypes.c_uint32),
        ("fFrameRate",            ctypes.c_float),
        ("nAudioBitrate",         ctypes.c_uint32),
        ("nFrameNum",             ctypes.c_uint32),
        ("nTimeStamp",            ctypes.c_uint32),  # 容器层时间戳（可能有误）
        ("stGlobalTime",          FC_GLOBAL_TIME),   # ★ 设备绝对时钟（准确）
        ("pData",                 ctypes.c_void_p),
        ("nDataLen",              ctypes.c_uint32),
        ("bLastPacket",           ctypes.c_bool),
        ("bFirstPacket",          ctypes.c_bool),
        ("nDataType",             ctypes.c_uint32),  # 1=video 2=audio
        ("enSystemFormat",        ctypes.c_int),
        ("enCodecType",           ctypes.c_int),
        ("nReserved",             ctypes.c_uint32 * 8),
    ]


DETAILED_CB_FUNC = ctypes.CFUNCTYPE(
    None,
    ctypes.POINTER(FC_DETAILED_CB_INFO),
    ctypes.c_void_p,
)

FC_OK = 0


class HikFormatConverter:
    """
    封装 FormatConversion.dll，将 HIK 格式分段转为标准 MP4。

    核心改进（相比原 SystemTransform 调用）：
      1. FC_SetGlobalTime  → 告知 SDK 该分段的 NVR 真实起始时刻
      2. FC_ReSetTimeStamp → 强制音频时间戳对齐到视频时间戳
      3. FC_RegisterDetailedCB → 记录每帧的设备绝对时钟供后续校验
    """

    def __init__(self, dll_dir: str):
        # 先加载依赖项（顺序重要）
        for dep in ["hpr.dll", "hlog.dll", "welsenc.dll",
                    "HWEncode.dll", "HWTranscode.dll"]:
            dep_path = os.path.join(dll_dir, dep)
            if os.path.exists(dep_path):
                ctypes.WinDLL(dep_path)

        dll_path = os.path.join(dll_dir, "FormatConversion.dll")
        self._dll = ctypes.WinDLL(dll_path)
        self._setup_prototypes()
        logger.info("FormatConversion.dll loaded, version=0x%08X",
                    self._dll.FC_GetSDKVersion())

    def _setup_prototypes(self):
        d = self._dll
        vp = ctypes.c_void_p

        d.FC_GetSDKVersion.argtypes  = []
        d.FC_GetSDKVersion.restype   = ctypes.c_uint32

        d.FC_CreateHandle.argtypes   = []
        d.FC_CreateHandle.restype    = vp

        d.FC_DestroyHandle.argtypes  = [vp]
        d.FC_DestroyHandle.restype   = ctypes.c_int

        d.FC_SetGlobalTime.argtypes  = [vp,
                                         ctypes.POINTER(FC_GLOBAL_TIME),
                                         ctypes.c_uint32]
        d.FC_SetGlobalTime.restype   = ctypes.c_int

        d.FC_ReSetTimeStamp.argtypes = [vp]
        d.FC_ReSetTimeStamp.restype  = ctypes.c_int

        d.FC_RegisterDetailedCB.argtypes = [vp, DETAILED_CB_FUNC, vp]
        d.FC_RegisterDetailedCB.restype  = ctypes.c_int

        d.FC_Start.argtypes   = [vp, ctypes.c_char_p, ctypes.c_char_p]
        d.FC_Start.restype    = ctypes.c_int

        d.FC_GetProgress.argtypes = [vp, ctypes.POINTER(ctypes.c_float)]
        d.FC_GetProgress.restype  = ctypes.c_int

        d.FC_Stop.argtypes    = [vp]
        d.FC_Stop.restype     = ctypes.c_int

    def convert_segment(
        self,
        input_path: str,
        output_path: str,
        segment_start_dt: datetime,
        timeout_sec: float = 600.0,
        frame_log_path: str | None = None,
    ) -> bool:
        """
        转封装单个 HIK 分段为标准 MP4。

        segment_start_dt：该分段在 NVR 上的真实起始时刻，
                          来自下载 API 中的 struStartTime，必须传入。
        frame_log_path：可选，记录每帧的设备绝对时钟，用于后续 PTS 校验。
        """
        h = self._dll.FC_CreateHandle()
        if not h:
            raise RuntimeError("FC_CreateHandle returned NULL")

        frame_log: list[dict] = []

        @DETAILED_CB_FUNC
        def _frame_cb(p_info, user):
            info = p_info.contents
            gt   = info.stGlobalTime
            frame_log.append({
                "track":     info.nTrackIndex,
                "type":      info.nDataType,
                "frame_num": info.nFrameNum,
                "pts_ms":    info.nTimeStamp,
                "abs_time":  (f"{gt.sYear:04d}-{gt.sMonth:02d}-{gt.sDay:02d} "
                              f"{gt.sHour:02d}:{gt.sMinute:02d}:{gt.sSecond:02d}"
                              f".{gt.sMs:03d}"),
            })

        try:
            # 注册逐帧回调（必须在 Start 前）
            self._dll.FC_RegisterDetailedCB(h, _frame_cb, None)

            # ★ 设置该分段的 NVR 真实起始时刻
            gt = FC_GLOBAL_TIME(
                sYear   = segment_start_dt.year,
                sMonth  = segment_start_dt.month,
                sDay    = segment_start_dt.day,
                sHour   = segment_start_dt.hour,
                sMinute = segment_start_dt.minute,
                sSecond = segment_start_dt.second,
                sMs     = segment_start_dt.microsecond // 1000,
            )
            ret = self._dll.FC_SetGlobalTime(h, ctypes.byref(gt), 1)
            if ret != FC_OK:
                logger.warning("FC_SetGlobalTime ret=0x%08X (non-fatal)", ret)

            # ★ 强制音频时间戳对齐到视频时间戳
            ret = self._dll.FC_ReSetTimeStamp(h)
            if ret != FC_OK:
                logger.warning("FC_ReSetTimeStamp ret=0x%08X (non-fatal)", ret)

            # 启动转封装
            ret = self._dll.FC_Start(
                h,
                input_path.encode("gbk"),
                output_path.encode("gbk"),
            )
            if ret != FC_OK:
                raise RuntimeError(f"FC_Start failed: 0x{ret:08X}")

            # 等待完成
            deadline = time.time() + timeout_sec
            while time.time() < deadline:
                pct = ctypes.c_float(0.0)
                self._dll.FC_GetProgress(h, ctypes.byref(pct))
                if pct.value >= 100.0:
                    break
                time.sleep(0.3)
            else:
                raise TimeoutError(f"Timeout after {timeout_sec}s: {input_path}")

            if frame_log_path and frame_log:
                with open(frame_log_path, "w", encoding="utf-8") as f:
                    json.dump(frame_log, f, ensure_ascii=False, indent=2)

            logger.info("Converted OK: %s → %s", input_path, output_path)
            return True

        finally:
            self._dll.FC_Stop(h)
            self._dll.FC_DestroyHandle(h)
```

```python
"""
hik_merge.py
使用 HmMerge.dll 合并多个已转封装的 MP4 分段。
与 VSPlayer 使用完全相同的合并引擎。
"""
import ctypes
import logging
import os

logger = logging.getLogger(__name__)


class HikMerger:
    """
    封装 HmMerge.dll 合并引擎。
    注意：HmMerge.dll 无公开头文件，参数结构为逆向推断，
    建议先在测试视频上验证返回码后再用于生产。
    """
    MERGE_STYLE_NORMAL = 0
    MERGE_STYLE_EE     = 1

    def __init__(self, dll_dir: str):
        for dep in ["hpr.dll", "hlog.dll"]:
            p = os.path.join(dll_dir, dep)
            if os.path.exists(p):
                ctypes.WinDLL(p)
        self._dll = ctypes.WinDLL(os.path.join(dll_dir, "HmMerge.dll"))
        logger.info("HmMerge.dll loaded: version=0x%08X",
                    self._dll.HM_GetVersion())

    def merge(self, input_paths: list[str], output_path: str) -> bool:
        h = self._dll.HM_CreateHandle()
        if not h:
            raise RuntimeError("HM_CreateHandle returned NULL")
        try:
            self._dll.HM_SetMergeStyle(h, self.MERGE_STYLE_NORMAL)

            # 构建以 \0 分隔、\0\0 结尾的文件列表
            file_list_buf = ("\0".join(input_paths) + "\0\0").encode("gbk")

            ret = self._dll.HM_Merge(
                h,
                ctypes.c_char_p(file_list_buf),
                ctypes.c_uint32(len(input_paths)),
                output_path.encode("gbk"),
            )
            if ret != 0:
                logger.error("HM_Merge failed: ret=0x%08X", ret)
                return False

            logger.info("HmMerge OK: %d segments → %s", len(input_paths), output_path)
            return True
        finally:
            self._dll.HM_DestroyHandle(h)


class FileEditMerger:
    """
    封装 FileEdit.dll，当 HmMerge 接口不稳定时作为备选。
    FileEdit.dll 的导出函数有明确的调用语义：AddToMergeList → MergeFileInList。
    """

    def __init__(self, dll_dir: str):
        for dep in ["hpr.dll", "hlog.dll"]:
            p = os.path.join(dll_dir, dep)
            if os.path.exists(p):
                ctypes.WinDLL(p)
        self._dll = ctypes.WinDLL(os.path.join(dll_dir, "FileEdit.dll"))

    def merge(self, input_paths: list[str], output_path: str) -> bool:
        h = self._dll.FILE_EDIT_CreateHandle()
        if not h:
            raise RuntimeError("FILE_EDIT_CreateHandle returned NULL")
        try:
            self._dll.FILE_EDIT_ClearMergeList(h)
            for path in input_paths:
                ret = self._dll.FILE_EDIT_AddToMergeList(
                    h, path.encode("gbk"))
                if ret != 0:
                    raise RuntimeError(
                        f"FILE_EDIT_AddToMergeList failed: {path}, ret=0x{ret:08X}")

            ret = self._dll.FILE_EDIT_MergeFileInList(
                h, output_path.encode("gbk"))
            if ret != 0:
                raise RuntimeError(
                    f"FILE_EDIT_MergeFileInList failed: ret=0x{ret:08X}")
            return True
        finally:
            self._dll.FILE_EDIT_Stop(h)
            self._dll.FILE_EDIT_DestroyHandle(h)
```

### 第三步：改进 ffmpeg.py 的检测和兜底修复逻辑

完成前两步后，ffmpeg.py 的角色从"主要修复者"变为"兜底检测和处理者"，只处理 SDK 仍无法覆盖的残余问题。

#### 3.3.1 新增段内 PTS 正向大跳变检测

HIK 最常见的段内问题是 PTS **正向大跳变**（如从 100s 跳到 5100s），现有代码只检测负向跳变，需补充：

```python
def probe_pts_large_gap(path: str, stream_selector: str = "v:0",
                         gap_threshold_sec: float = 2.0) -> dict:
    """检测 PTS 正向大跳变（HIK 段内常见异常）。"""
    cmd = [
        _ffprobe_bin(), "-v", "error",
        "-select_streams", stream_selector,
        "-show_packets",
        "-show_entries", "packet=pts_time,duration_time",
        "-of", "csv=p=0", path,
    ]
    try:
        out = subprocess.check_output(cmd, stderr=subprocess.DEVNULL,
                                      text=True, encoding="utf-8", errors="ignore")
    except Exception:
        return {"large_gap_count": 0, "max_gap_sec": 0.0}

    large_gap_count, max_gap_sec = 0, 0.0
    prev_pts, prev_dur = None, 0.033

    for line in (out or "").splitlines():
        parts = line.split(",")
        if len(parts) < 2:
            continue
        try:
            pts = float(parts[0])
            dur = float(parts[1]) if parts[1] not in ("", "N/A") else prev_dur
        except Exception:
            continue
        if prev_pts is not None:
            gap = pts - prev_pts
            if gap > gap_threshold_sec:
                large_gap_count += 1
                max_gap_sec = max(max_gap_sec, gap)
        prev_pts, prev_dur = pts, dur

    return {"large_gap_count": int(large_gap_count),
            "max_gap_sec": float(max_gap_sec)}
```

在 `rebuild_zero_based_timeline` 现有检测之后加入：

```python
    import json as _json

           "-analyzeduration", "100M", "-probesize", "100M",
           "-print_format", "json", "-show_streams", path]
    out = subprocess.check_output(cmd, stderr=subprocess.DEVNULL,
                                  text=True, encoding="utf-8", errors="ignore")
    meta = {"video_codec": None, "audio_codec": None, "sample_rate": None}
    for s in _json.loads(out).get("streams", []):
        ct = s.get("codec_type", "")
        if ct == "video" and not meta["video_codec"]:
            meta["video_codec"] = s.get("codec_name")
        elif ct == "audio" and not meta["audio_codec"]:
            meta["audio_codec"] = s.get("codec_name")
            meta["sample_rate"] = s.get("sample_rate")
    return meta


def validate_segments_before_merge(segment_paths: list[str]) -> list[str]:
    """返回与第一段格式不兼容的分段路径列表。合并前必须调用。"""
    if not segment_paths:
        return []
    metas = [(p, probe_segment_meta(p)) for p in segment_paths]
    ref = metas[0][1]
    return [p for p, m in metas[1:]
            if (m["video_codec"] != ref["video_codec"]
                or m["audio_codec"] != ref["audio_codec"]
                or m["sample_rate"] != ref["sample_rate"])]
```

#### 3.3.3 ffmpeg 兜底合并（加 duration 指令防止 PTS 累积漂移）

```python
def merge_with_ffmpeg_duration(segment_paths: list[str],
                                output_path: str,
                                durations_sec: list[float] | None = None) -> None:
    """
    用 ffmpeg concat demuxer 合并，指定每段精确时长，防止 PTS 累积漂移。
    durations_sec 最好来自 NVR API 的 struStartTime/struStopTime 差值。
    """
    import tempfile, json as _json, os as _os

    if not durations_sec:
        durations_sec = []
        for p in segment_paths:
            cmd = [_ffprobe_bin(), "-v", "quiet",
                   "-print_format", "json", "-show_format", p]
            out = subprocess.check_output(cmd, stderr=subprocess.DEVNULL,
                                          text=True, encoding="utf-8", errors="ignore")
            dur = float(_json.loads(out).get("format", {}).get("duration") or 0)
            durations_sec.append(dur)

    lines = []
    for path, dur in zip(segment_paths, durations_sec):
        lines.append(f"file '{Path(path).resolve()}'")
        if dur > 0:
            lines.append(f"duration {dur:.6f}")

    with tempfile.NamedTemporaryFile(mode="w", suffix=".txt",
                                     delete=False, encoding="utf-8") as tf:
        tf.write("\n".join(lines) + "\n")
        concat_file = tf.name

    try:
        cmd = [
            "ffmpeg", "-y",
            "-f", "concat", "-safe", "0",
            "-i", concat_file,
            "-c", "copy",
            "-avoid_negative_ts", "make_zero",
            "-movflags", "+faststart",
            output_path,
        ]
        result = subprocess.run(cmd, capture_output=True, text=True,
                                encoding="utf-8", errors="ignore")
        if result.returncode != 0:
            raise RuntimeError(f"ffmpeg merge failed:\n{result.stderr[-2000:]}")
    finally:
        _os.unlink(concat_file)
```

## 七、历次分析的错误结论汇总（已全部更正）

| 错误结论 | 正确结论 |
|---|---|
| 推荐使用 `.so` 文件（libFormatConversion.so 等） | 服务端是 Windows，应使用 `.dll` 文件 |
| `SYSTRANS_ResetPackInfo(bTimestampSync=1)` 是关键调用 | 该函数**不存在**于 `SystemTransform.dll` 导出表，正确 API 是 `FormatConversion.dll` 中的 `FC_ReSetTimeStamp` |
| VSPlayer DLL 不可在服务端使用 | 服务端是 Windows，VSPlayer 的 DLL **可以直接提取使用** |
| HmMerge.dll 只能用于播放场景 | Windows 服务端可以用 Python ctypes 直接调用，用于服务端视频合并 |
