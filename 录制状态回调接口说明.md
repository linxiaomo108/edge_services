# 移动录课 - 录制状态回调接口说明

> 提供方：教育智慧校园服务端
> 调用方：三方边缘服务
> 用途：边缘服务在录制任务状态发生变化（结束 / 取消 / 失败 / 中断）时，回调本接口，将最终的录制结果与文件信息同步给服务端。

---

## 1. 接口信息

| 项目 | 内容 |
| --- | --- |
| 请求方式 | `POST` |
| 接口地址 | `{callbackBaseUrl}/api/v1/record-task/callback` |
| Content-Type | `application/json` |
| 字符编码 | `UTF-8` |

> `callbackBaseUrl` 即边缘服务当前配置中的服务端基础地址，一般取 `serverAddress`（例如 `http://192.168.9.83:8090`）。边缘服务会自动拼成 `{callbackBaseUrl}/api/v1/record-task/callback`，服务端不需要在 `start` 接口里再传 `callbackUrl`。

---

## 2. 调用时机

录制任务出现以下状态变化时回调：

- `recording`：录制已成功开始
- `finished`：录制正常结束
- `cancelled`：录制被取消
- `failed`：录制失败
- `interrupted`：录制中断

---

## 3. 请求参数（Request Body）

| 字段 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| taskId | string | 是 | 录课任务ID（即 start 请求中下发的 taskId） |
| classroomId | string | 否 | 教室ID |
| cameraId | string | 否 | 摄像机ID |
| status | string(enum) | 是 | 录制状态：`recording` / `finished` / `cancelled` / `failed` / `interrupted` |
| stopReason | string \| null | 否 | 停止原因，如 `manual`、`cancel`、`auto_timeout` |
| playUrl | string \| null | 否 | 文件播放地址 |
| outputDir | string | 否 | 输出目录（服务端暂不存储） |
| m3u8Path | string | 否 | 本地 index.m3u8 路径 |
| segmentCount | int | 否 | HLS 分片数量 |
| fileSize | int | 否 | 文件大小（字节） |
| duration | float | 否 | 最终可播放时长（秒） |
| codec | string | 否 | 编码格式，常见值 `h264`、`hevc` |
| format | string | 否 | 封装格式（服务端暂不存储） |
| startTime | string | 否 | 开始时间，格式 `yyyy-MM-dd HH:mm:ss` 或 ISO-8601 |
| finishTime | string | 否 | 完成时间，格式 `yyyy-MM-dd HH:mm:ss` 或 ISO-8601 |
| errorMessage | string \| null | 否 | 错误信息（失败/中断时填写） |

### 请求示例

```json
{
  "taskId": "15",
  "classroomId": "1729445100872728578",
  "cameraId": "7",
  "status": "recording",
  "stopReason": null,
  "playUrl": "https://xxx.com/record/15/index.m3u8",
  "outputDir": "/data/record/15",
  "m3u8Path": "/data/record/15/index.m3u8",
  "segmentCount": 0,
  "fileSize": 0,
  "duration": 0.0,
  "codec": null,
  "format": "hls",
  "startTime": "2026-06-12 09:00:00",
  "finishTime": null,
  "errorMessage": null
}
```

### 结束回调示例

```json
{
  "taskId": "15",
  "classroomId": "1729445100872728578",
  "cameraId": "7",
  "status": "finished",
  "stopReason": "manual",
  "playUrl": "https://xxx.com/record/15/index.m3u8",
  "outputDir": "/data/record/15",
  "m3u8Path": "/data/record/15/index.m3u8",
  "segmentCount": 120,
  "fileSize": 524288000,
  "duration": 1800.0,
  "codec": "h264",
  "format": "hls",
  "startTime": "2026-06-12 09:00:00",
  "finishTime": "2026-06-12 09:30:00",
  "errorMessage": null
}
```

---

## 4. 服务端处理逻辑

1. 通过 `taskId` 匹配录课任务与录制文件记录。
2. 根据 `status` 更新任务状态：
   - `recording` → `RECORDING`
   - `finished` → `FINISHED`
   - `cancelled` → `CANCELLED`
   - `failed` → `FAILED`
   - `interrupted` → `FAILED`
3. 更新文件信息（`playUrl`、`m3u8Path`、`segmentCount`、`fileSize`、`duration`、`codec`、`startTime`、`finishTime`），其中 `duration` 会四舍五入为整数秒存储。
4. `stopReason`、`errorMessage` 同步记录到任务上。

> 仅回传的非空字段会被更新，未携带的字段保持原值。

---

## 5. 响应结果（Response）

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| code | string | `SUCCESS` 表示已接收并处理成功；`FAILED` 表示处理失败 |
| message | string | 处理结果描述 |

### 成功响应示例

```json
{
  "code": "SUCCESS",
  "message": "回调处理成功"
}
```

### 失败响应示例

```json
{
  "code": "FAILED",
  "message": "错误描述"
}
```

> 边缘服务收到 `code=SUCCESS` 即视为本次回调已确认；若收到 `FAILED` 或请求超时，建议按重试策略重发。

---

## 6. 备注

- 接口具备幂等性：同一任务多次回调，以最新一次回传为准进行更新。
- 时间字段建议统一使用 `yyyy-MM-dd HH:mm:ss`（东八区）。
